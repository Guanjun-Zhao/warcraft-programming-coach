## 结构化规则（版四：终极版）

### 地图与对象

- 城市 `1 … N`（N ≤ 20）；红司令部为 **0**，蓝司令部为 **N+1**。司令部有生命元造兵。
- 武士编号规则同前：双方第 n 个武士编号均为 n。
- 生命值战斗中变化；≤0 视为 0 则死亡。
- 武器三种：`sword`、`bomb`、`arrow`（编号 0、1、2）。

### 战斗（基础，版四模型）

- 武士向对方司令部移动；同城最多一红一蓝；发生战斗时 **一方主动进攻一次**，受损方可 **反击一次**（题面给定伤害公式：含 sword 加成；反击伤害为攻击力一半去尾 + sword 等）。
- 击杀敌人可得司令部奖励生命元（若司令部有足够生命元；奖励优先级：**距敌方司令部更近者优先**）。击杀后还可收取城市中的战利品生命元；**先完成全部奖励，再回收城市生命元**；奖励不足则不事后补发。
- **平局**（双方幸存）：双方**不能**取走该城市生命元。
- **bomb 同归于尽**：题面规定 **不算一场战斗**；双方不取城市生命元；**不影响**旗帜逻辑（见题面细则）。

### 先手与旗帜（城市）

- 插红旗城市 + **奇数**编号且无旗城市 → **红**主动进攻；插蓝旗 + **偶数**且无旗 → **蓝**主动进攻。
- **连续两场**均为同一方击杀（中间无战斗的时刻可跨过；**平局打断连续性**）→ 该城插上胜方旗；原敌旗可被替换。旗帜持久直至被敌更换。

### 武器特性（摘要）

- **sword**：初始攻击力 = 拥有者攻击力的 20%（去尾）；每经历一次战斗（主动或反击）后攻击力变为本次战斗前的 **80%**（去尾）；为 0 则视为失去 sword；降生时 sword 攻击力为 0 视为无 sword。
- **arrow**：攻击力 R；若**下一步要进入的城市**有敌人则可 **放箭**（不进入该城也可射下一城敌人；**不能**射对方司令部内敌人）；用 **3** 次耗尽；两相邻武士可同时互射致死。
- **bomb**：战斗开始前若判断自己将被杀死可与敌同归于尽；题面强调「不预测对方 bomb」。

### 武士特性（摘要）

- **dragon**：一件武器，降生 **n%3**；士气 = 降生后司令部剩余生命元 / 造 dragon 所需生命元；主动进攻战斗结束未死且士气 >0.8 可欢呼；胜负改变士气 ±0.2（细节见题面）。
- **ninja**：两件武器 **n%3** 与 **(n+1)%3**；**从不反击**。
- **iceman**：一件 **n%3**；每前进 **两步**，在第 **2** 步完成时生命 −9、攻击 +20；若减 9 会使生命 ≤0 则不减而变为 **1**（不会因走路而死）。
- **lion**：忠诚度初值 = 降生后司令部剩余生命元；每场 **未能杀死敌人** 的战斗忠诚度 −K；≤0 则逃离（已达敌方司令部的 lion 不逃）；己方司令部也可能逃；战前生命若战死转移给对手（「战斗前」时刻以题面为准）。
- **wolf**：降生无武器；**战斗获胜**可缴获敌人武器（已有则不重复缴）；缴获武器保持已磨损状态。

### 时间轴（每小时）

题面规定的分钟事件（需严格对齐顺序与题意）：

- **:00** 双方各降生一名武士（规则允许时）。
- **:05** lion 逃跑。
- **:10** 所有武士向敌司令部方向前进一步。
- **:20** 每个城市产出 **10** 个生命元（留在城市直至被取走）。
- **:30** 若某城市 **仅一名** 武士，其取走该城全部生命元并立即送回所属司令部。
- **:35** 拥有 arrow 的武士放箭（放箭 **不是**战斗；放箭地点算箭发出城市）。
- **:38** 拥有 bomb 的武士判断是否同归于尽。
- **:40** 有两名武士的城市发生战斗；若敌人在 **:35** 已被箭射死，仍视为发生战斗，胜者享受胜利后流程（题面：欢呼判定仍与主动进攻轮次有关）。
- **:50** 司令部报告拥有生命元数量。
- **:55** 武士报告武器情况。

### 制造顺序与造兵（版四）

- 红：`iceman` → `lion` → `wolf` → `ninja` → `dragon`；蓝：`lion` → `dragon` → `ninja` → `iceman` → `wolf`。
- 生命元不足：等到获得足够生命元后的 **第一个整点** 再制造**轮到的那种**武士（**不顺延到其他兵种**）。

### 战争结束

- 武士抵达对方司令部后停留。
- 任一方司令部存在 **2 名敌人** → 该司令部被占领 → **战争结束**；结束后不再发生后续事件，但与占领**同时刻**的事件仍须输出。

### 输出格式要点

- 首行每组：`Case n:`（**有空格**）。
- 时间：`HHH:MM`。
- 事件类型与完整字符串模板见题面（降生、`Its morale` / `Its loyalty`、前进、放箭、bomb、进攻、反击、战死、欢呼、**赚取生命元** `earned … for his headquarter`、升旗、抵达、占领、司令部报告、武器报告等）。
- **武器报告（:55）**：次序为 **arrow、bomb、sword**；格式如 `has arrow(2),bomb,sword(23)` 或 `has no weapon`（与版三「sword、bomb、arrow」次序 **不同**，切勿混用）。
- **输出顺序**：按时间；同刻自西向东；前进发生在目的地；同一战斗内多事件按题面给定编号顺序；两武士同时抵达同城先红后蓝；占领与抵达同时刻则先抵达后占领（题面规则）。

### 输入格式

- 第一行 t。
- 每组三行：  
  - `M N R K T`：生命元 M、城市数 N、arrow 攻击力 R、lion 忠诚度衰减 K、截止时刻 T（分钟，含 T）。  
  - 五种初始生命值；五种攻击力（顺序均为 dragon、ninja、iceman、lion、wolf）。

### 精度与实现

- 大量去尾取整与分数运算；优先使用整数运算书写比例关系，避免不必要浮点误差。
